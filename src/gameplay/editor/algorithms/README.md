# 关卡编辑器算法扩展

编辑器算法采用独立策略模块。算法可以拥有完全不同的参数结构和路径逻辑，不要求与算法1共享数值字段。

## 数据结构

关卡 JSON 使用以下字段保存算法选择：

```json
{
  "algorithm": {
    "id": "algorithm-1",
    "parameters": {
      "topology": "board-shape",
      "pathMode": "single-stroke",
      "targetCrossings": 0
    }
  }
}
```

旧关卡缺少 `algorithm` 时按算法1读取。

新建编辑器关卡默认使用算法2：最大交叉数量 `20`、拐弯概率 `40%`、隐藏比例 `50%`。

六边形蜂窝棋盘不产生交叉，算法1会强制将 `targetCrossings` 解析并保存为 `0`。为兼容旧关卡，JSON 字段仍叫 `targetCrossings`，编辑器中统一按“最大交叉数量”解释，不会为了凑满该数值主动制造交叉。

算法2在大多数步骤按“拐弯概率”随机打乱候选方向，少量步骤使用低自由度安全引导，死局、断路或超过交叉上限时通过回溯改走其他方向。每次生成多条完整路径后随机选择一条；若随机搜索未在预算内完成，则使用稳定搜索作为后备。随后随机翻转仍能合法重连的路径片段，每次变换都重新验证相邻关系和交叉上限，用来打散局部区域的填充顺序。确定路径后再生成固定隐藏数字。它会搜索当前显示数字约束下的第二条完整解；若存在，就显示两条完整解首次分歧位置的正确数字，并继续验证，直到完整解唯一。随后只在不会重新引入第二解的位置补回隐藏数字。若目标隐藏比例与唯一解冲突，优先保证无纯运气分叉。

## 新增算法

1. 在 `types.ts` 定义独立参数接口和 selection，并加入 `EditorAlgorithmSelection` 联合类型。
2. 新建算法模块，实现默认参数和执行逻辑。
3. 在 `registry.ts` 注册名称、说明、兼容解析和运行分派。
4. 在 `parameterEditors.ts` 添加该算法专属的 DOM 参数面板。
5. 算法运行只接收 `EditorAlgorithmContext`，不要直接访问控制器、视图或 localStorage。

算法模块负责规则，参数编辑器负责 DOM，`LevelEditorModel` 只保存选择并调用注册表。
